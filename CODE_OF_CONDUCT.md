# Code of Conduct

## Our standard

We want this project to be respectful, practical, and technically useful.

Welcome behavior includes being constructive, protecting credentials and device identifiers, accepting corrections, and focusing criticism on code and documentation rather than people.

Unacceptable behavior includes harassment, discriminatory language, publishing private information, posting live SIP credentials or onboarding tokens, and intentionally misleading others about test results or device support.

## Scope

This applies to issues, pull requests, reviews, discussions, and other project-managed spaces.

## Enforcement

Maintainers may edit or remove content and restrict participation when this code is violated. Report sensitive concerns privately to the repository owner rather than opening a public issue.
